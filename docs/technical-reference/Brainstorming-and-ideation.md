- **課題の理解**: Issue の説明やコメント、関連ドキュメントを読み、解決すべき問題を正確につかみます。

- **発散と調査**: ブレインストーミングで創造的なアイデアや解決策を出し、業界のベストプラクティス、ユーザーの嗜好、既存のデザインパターンなどを調査してアプローチに活かします。

- **コメントと議論**: GitHub Issue 内で初期の考えやコンセプト、デザインの方向性を共有し、メンテナーや他の貢献者、デザイナーと議論してフィードバックをもとにアイデアを磨きます。

- **コンセプトのスケッチ**: 必要に応じて紙やデジタルツールでラフ案を描き、詳細設計に入る前にアイデアを可視化します。

## ワイヤーフレームとコンセプトの作成

- **ワイヤーフレーム作成**: 採用したコンセプトに基づき、UI のレイアウト・構造・操作フローに集中した低忠実度のワイヤーフレームを作成します。

- **反復とブラッシュアップ**: ワイヤーフレームを GitHub Issue で共有し、コミュニティからフィードバックを集めて改善を重ね、懸念点や提案に対応します。
